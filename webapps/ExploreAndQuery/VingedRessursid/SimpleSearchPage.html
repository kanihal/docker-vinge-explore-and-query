This page is the entry in to the Data Maze. Look for the door! You have following options for how to get in:

Free text search

By searching for instances of things you know should exist in the data. This is free text search, like Google. From hit list select one instance and start browsing relations and explore the model.

Concept search

Searching for a concept or class of instances, then by selecting one class and listing the instances of it. Select one instance and start exploring.
Only concepts defined as owl:Class or rdfs:Class are listed.

Property search

Searching for a property (attribute or relation), then selecting one and listing pairs of subject-object where the selected property occurs. Select either one and start exploring.
Only properties defined as owl:DatatypeProperty or owl:ObjectProperty are listed.

Combination of above

Limit the text search with the concept and/or property.
When combining with concept then the object of a triple pattern is limited by the selected concept.
When combining with property then the the free text is searched in the subject literal that the selected property points to. In this case the selected property must be datatype property.


Enter Free Text string


You can combine words logically with AND, OR, NOT, NEAR using brackets
You can use wildcards after 4 character. You need to use quotes when using wildcards and logical operators. If you don't use quotes then AND is appllied between words
Example: Soccer player , ("heart" OR "cardi*") AND NOT ("alco*" OR "smok*") AND ("infect*" NEAR muscle)

NB! If your search string contains any unicode characters, you need to put them between quotes as well. Example: "München"

! Not all Triple stores/SPARQL endpoints support Free Text Search
